# Re:cord Feature Keep / Hide / Cut Matrix

기준 날짜: 2026-03-27  
작성 목적: 지금 있는 기능을 무엇으로 취급할지 빠르게 판단하기 위한 의사결정 표

결정 값:

- `KEEP`: 메인 제품으로 유지
- `KEEP, SHARPEN`: 유지하되 더 선명하게 다듬기
- `KEEP, HIDE`: 기능은 유지하지만 메인 메시지/동선에서는 뒤로 보냄
- `FREEZE`: 당장 확장하지 말고 현상 유지
- `SUNSET`: 제거 또는 통합 대상

## 7. 지금 바로 해야 할 정리 우선순위

### P1

- 공개 프로필 / 리뷰 / 공유를 메인 제품 코어로 확정
- README와 메인 카피를 현재 전략에 맞게 갱신
- `대표 후기`를 실제 모델/UX 개념으로 올릴지 결정
- OCR와 migration을 메인 메시지에서 내리고 보조 레이어로 정의

### P2

- PT / Kmong을 메인 전략이 아닌 실험 축으로 정리
- pricing에서 통계/커스터마이징보다 공개 링크 운영 가치를 앞세우기
- 결제/플랜 로직을 한 소스로 통합

### P3

- Reboot / Customize / customCss 같은 잔재를 정리
- migration request를 ticket description 문자열이 아닌 구조화 모델로 분리 검토
- demo fallback을 실제 runtime 코어에서 분리

## 8. 의사결정 원칙

새 기능이나 기존 기능의 우선순위를 정할 때는 아래 질문으로 판단한다.

1. 이 기능이 링크를 더 빨리 보내게 만드는가?
2. 이 기능이 고객이 30초 안에 신뢰를 느끼게 만드는가?
3. 이 기능이 대표 후기 3개와 작업 사진의 설득력을 높이는가?
4. 이 기능이 초기 wedge인 헤어디자이너에 더 직접적으로 도움이 되는가?
5. 이 기능이 제품 정체성을 더 선명하게 만드는가, 더 흐리게 만드는가?

위 질문에 대부분 `아니오`라면, 지금은 메인 기능이 아니다.
