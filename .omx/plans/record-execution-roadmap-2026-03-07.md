# Record Execution Roadmap (2026-03-07)

## Requirements Summary
- Reposition Record from review OCR tool to portable trust page for independent professionals.
- Preserve working functionality while improving positioning, conversion, trust semantics, and operational safety.
- Prioritize a 1-week shippable scope that creates visible business value.
- Define a 12-month roadmap without pretending all of it can be delivered immediately.

## Acceptance Criteria
- Landing clearly targets one wedge segment first.
- Public profile supports at least one real conversion action beyond passive viewing.
- Trust semantics are less misleading than today.
- Pricing/plan limits are internally consistent.
- Public review submission and review detail access are safer than today.
- Scope for week 1 fits a single focused sprint.

## Week 1 Priorities
1. Narrow positioning to hair stylists first.
2. Upgrade public profile into a trust page with CTA actions.
3. Ship a “Reboot Pack” onboarding flow for move/independence moments.
4. Fix trust/plan/security inconsistencies that can damage credibility.

## 12-Month Shape
- Q1: trust page and proof workflow
- Q2: mobile capture, analytics, team-lite
- Q3: proof graph and discovery beta
- Q4: lead routing / marketplace experiments

## Risks
- Brand promise outruns product truth.
- Schema drift breaks live features.
- OCR is mistaken for moat.
- Discovery is attempted before enough trusted data exists.

## Verification
- Manual walkthrough of landing -> signup -> upload -> publish -> share -> inquiry.
- Verify free/premium/pro behavior against pricing page.
- Verify public profile only shows intended trust signals.
